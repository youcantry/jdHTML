写项目时，往往会使用到很多小图片，一般情况下，
设计师会把一堆的小图片合成一个大图片，叫精灵图。

为什么要使用精灵图？
答：项目最后放到服务器上，在浏览器的地址栏中输入www.jd.com/index.html,
默认是请求服务器上的index.html文件，服务器就把你要的这个index.html文件返回
给浏览器，浏览器得到得index.html后，发现index.html中有很多<img src=""/> 
浏览器会根据img的src发起二次请求，如果你使用了很多的小图片，那么浏览器发发
起很多的二次请求，每一次请求都是耗时的，如果把一堆的小图
合成一个大图，只需要去请求一张大图就OK了，浏览器得到这个大图
后，根据坐标去找到大图中的某个小图，这样做的目的，是有利于前端页面的
性能优化。

一般情况下，得到的图片都是二倍图。

什么是二倍图？
答：设计的尺寸比实际的尺寸大一倍，如：logo 50*50
    设计师设计出来的是 100*100 

为什么把图片都搞成二倍图？
答：前面说过了  为了更加清晰地显示图片  
    至于原理，先这样记住。

------------------------------------
什么是绝对定位？
    绝对定位是根据参考点进行定位

如何设置参考点：
    子标签设置绝对定位，父标签设置相对定位。
    子绝父相    
    子绝父固

轮播图的原理：
    一个小盒子中装一个非常长的盒子，通过JS控制长盒子移动。

------------------------------------
在CSS中两个东西比较恶心：
1）浮动  
    一个盒子中的子元素都浮动了  父元素高度就塌陷了  
    解决：给父元素添加overflow:hidden
2）绝对定位 
    一个盒子绝对定位 也会造成父元素高度的塌陷
    解决：手动加上一个高度
